## 1. 목표

wave를 클리어하면 별도 상점 화면으로 이동하지 않는다. 전투는 멈추고, 밤하늘 전장 배경은 그대로 유지한 채 무기 아이템 상세 패널 4개를 2x2로 화면 전체에 배치한다. 플레이어는 캐릭터를 원하는 패널 칸으로 이동시켜 멈추고, coin을 1개씩 소모하며 `???`로 잠긴 스탯을 직접 공개한다.

핵심 목표:

- 대량 드랍 아이템을 하나하나 비교하는 스트레스를 줄인다.
- 한 wave마다 4개의 제한된 기회를 제공한다.
- 완성된 아이템을 바로 고르는 것이 아니라, 숨겨진 랜덤 스탯을 하나씩 열어보는 재미를 만든다.
- 기본 스탯은 저렴하게 공개해 아이템 획득까지 연결하고, 부가 옵션은 비싸게 만들어 coin sink로 쓴다.
- 유저가 직접 옵션을 reroll하고 감정하는 듯한 감각을 준다.

## 3. 전체 흐름

```text
Wave Clear
  -> coin 지급
  -> 전투 정지, 적 소환 정지
  -> 위험 투사체와 남은 적 정리
  -> 미감정 WeaponInstance 4개 생성
  -> 2x2 아이템 상세 패널 표시
  -> 모든 기본 스탯과 부가 옵션은 ???로 잠김
  -> 플레이어가 원하는 패널 영역으로 이동
  -> 정지 상태에서 coin 1개씩 소모, 현재 ??? 게이지 충전
  -> 기본 스탯 전체 언락 시 해당 아이템 획득 가능
  -> 부가 옵션은 언락한 만큼만 최종 아이템에 적용
  -> 인벤토리 보관 또는 1-3번 무기 슬롯에 장착
  -> 슬롯 순서 정비
  -> 다음 wave 시작
```

### 3.1 등장 타이밍

| 구간 | 등장 규칙 | 의도 |
| --- | --- | --- |
| Wave 1-2 | 미등장 또는 튜토리얼 1회 | 초반 전투 학습 우선 |
| Wave 3-5 | wave 클리어 시 60% 확률 | 시스템 소개 |
| Wave 6-10 | wave 클리어 시 80% 확률 | 빌드 선택 본격화 |
| Wave 11+ | wave 클리어 시 100% 확률 | 성장 루프 고정 |
| 보스/엘리트 wave 후 | 항상 등장, 희귀도와 affix 수 보정 | 큰 전투 후 확실한 보상감 |

### 3.2 화면 구성

화면 전체를 2x2 패널로 나눈다.

```text
┌────────────────────┬────────────────────┐
│ Panel A            │ Panel B            │
│ 미감정 무기         │ 미감정 무기         │
│ ??? 0/4            │ ??? 0/5            │
├────────────────────┼────────────────────┤
│ Panel C            │ Panel D            │
│ 미감정 무기         │ 미감정 무기         │
│ ??? 0/4            │ ??? 0/6            │
└────────────────────┴────────────────────┘
```

패널은 디아블로식 아이템 설명 패널처럼 구성한다.

- 상단: 등급, 미감정 이름, 약한 힌트
- 중단: 기본 스탯 `???` row
- 하단: 부가 옵션 `???` row
- 우측 또는 하단: 현재 coin, 획득 가능 상태, 장착/보관 버튼
- 패널 테두리: 플레이어가 해당 영역 안에 있고 정지 중인지 표시

## 4. 핵심 규칙

### 4.1 전투 정지

감정 phase에 들어가면 전투 위험을 모두 멈춘다.

- 적 소환 중지
- 자동 공격 중지
- 무기 쿨타임 진행 중지
- 남은 적 제거 또는 퇴각 처리
- 위험 투사체, 장판, 충돌 피해 제거
- 플레이어 이동은 유지
- 카메라는 기존 밤하늘 전장 배경 유지

### 4.2 패널 진입과 정지 판정

플레이어 캐릭터의 위치로 활성 패널을 결정한다.

```text
캐릭터가 Panel A 영역 안에 있음
  -> 이동 중이면 대기
  -> 정지 상태가 일정 시간 유지되면 충전 시작
  -> Panel A의 다음 잠금 row를 active row로 선택
```

정지 판정 권장값:

- 이동 입력 없음
- 실제 속도 `0` 또는 아주 낮은 임계값 이하
- 최소 정지 시간 0.15-0.25초

이동 중에는 coin이 빠지지 않고 게이지도 차지 않는다.

### 4.3 게이지와 coin 소모

각 `???` row는 `현재/필요 coin` 형태의 게이지를 가진다.

```text
??? 기본 스탯 1    0/4
??? 기본 스탯 2    0/5
??? 부가 옵션 1    0/10
```

소모 규칙:

- coin은 row가 완성될 때 한 번에 빠지는 것이 아니다.
- 정지 중 일정 tick마다 `coin -1`, `revealProgress +1`을 처리한다.
- `revealProgress == revealCost`가 되면 row를 공개한다.
- 공개된 row는 다시 잠기지 않는다.
- 이미 공개된 row에 사용한 coin은 환불하지 않는다.

### 4.4 중단과 초기화

진행 중 row의 게이지만 초기화한다.

| 상황 | 처리 |
| --- | --- |
| 패널 안에서 이동 시작 | 현재 진행 중 row `revealProgress = 0` |
| 다른 패널로 이동 | 이전 패널의 진행 중 row `revealProgress = 0` |
| 패널 밖으로 이탈 | 현재 진행 중 row `revealProgress = 0` |
| 이미 공개된 row | 유지 |
| 이미 공개한 기본 스탯 | 유지 |
| 이미 공개한 부가 옵션 | 유지 |

### 4.5 coin 부족

coin이 부족해도 연출은 동일하게 시작한다.

```text
부가 옵션 3 비용: 24
현재 coin: 9
정지 유지
  -> 0/24, coin 9
  -> 1/24, coin 8
  ...
  -> 9/24, coin 0
  -> coin 부족으로 정지
```

coin이 0이 되면 충전이 멈춘다. 이 상태에서 패널을 벗어나면 `9/24` 진행도는 0으로 돌아간다. 이미 공개된 row는 유지한다.

## 5. 아이템 후보 생성

감정 phase가 시작될 때 4개의 `WeaponInstance`를 미리 생성한다. 실제 값은 이미 결정되어 있지만, UI에는 공개된 row만 보여준다.

```text
RevealPanel = hidden WeaponInstance + RevealRow[]
```

후보 생성 순서:

1. waveIndex와 seed를 받는다.
2. 후보 4개의 rarity를 뽑는다.
3. 원소와 공격 방식을 조합한다.
4. 기본 스탯을 굴린다.
5. 부가 옵션 4-6개를 미리 굴린다.
6. 기본 스탯 row와 부가 옵션 row를 만든다.
7. 모든 row를 `revealed: false`로 시작한다.
8. 2x2 패널에 배치한다.

### 5.1 원소와 공격 방식

무기는 `기본 원소 1개 + 공격 방식 1개`로 만든다.

| 원소 | 역할 |
| --- | --- |
| `fire` | 광역 피해, 폭발, 지속 피해 |
| `water` | 둔화, 제어, 안정성 |
| `electric` | 연쇄 피해, 빠른 공격, 순간 폭딜 |
| `sand` | 탄막, 방어적 공간 제어, 장판 견제 |

| 공격 방식 | 역할 |
| --- | --- |
| `bolt` | 단일 투사체 |
| `fan` | 부채꼴 다중 발사 |
| `missile` | 느린 추적탄 |
| `meteor` | 지연 광역 낙하 |
| `rain` | 여러 투사체 낙하 |
| `beam` | 지속 관통 공격 |
| `orbit` | 플레이어 주변 회전 타격 |

### 5.2 기본 스탯

기본 스탯 row는 순서를 고정한다. 플레이어가 어떤 패널을 열어도 같은 순서로 정보를 얻어야 판단 스트레스가 낮다.

권장 순서:

1. 원소 + 공격 방식
2. 공격력
3. 발사 쿨타임
4. 공격 범위
5. 발사체 이동 속도 또는 공격 방식 전용 수치

기본 비용 예시:

| Row | 비용 | 공개 예시 |
| --- | ---: | --- |
| 기본 1 | 4 | `Fire / Missile` |
| 기본 2 | 5 | `공격력 19` |
| 기본 3 | 6 | `쿨타임 0.88s` |
| 기본 4 | 7 | `범위 480` |
| 기본 5 | 8 | `발사체 속도 342` |

모든 기본 스탯을 공개하면 아이템은 `획득 가능` 상태가 된다.

### 5.3 부가 옵션

부가 옵션은 4-6개를 생성한다. 뒤로 갈수록 비용이 크게 오른다.

| Affix row | 비용 예시 | 의도 |
| --- | ---: | --- |
| 부가 1 | 10 | 가볍게 열어보는 보너스 |
| 부가 2 | 16 | 선택 압력 시작 |
| 부가 3 | 24 | 좋은 후보에 더 투자할지 판단 |
| 부가 4 | 36 | 고가 coin sink |
| 부가 5 | 52 | 후반 투자처 |
| 부가 6 | 74 | 매우 좋은 후보 전용 투자 |

부가 옵션은 공개한 row만 최종 아이템에 붙는다. 공개하지 않은 옵션은 폐기한다.

## 6. 획득과 장착

### 6.1 획득 가능 조건

아이템 획득 조건:

- 해당 패널의 모든 기본 스탯 row가 공개됨
- 아직 `claimed` 상태가 아님
- 인벤토리에 빈 칸이 있거나 장착 교체 UI가 열릴 수 있음

기본 스탯을 모두 열기 전에는 부가 옵션이 좋아 보여도 획득할 수 없다.

### 6.2 최종 아이템 적용

획득 시점에 `WeaponInstance`를 최종화한다.

```text
finalWeapon.rolledStats = hiddenWeapon.rolledStats
finalWeapon.affixes = revealedAffixRows.map(row => row.affix)
```

예시:

```text
기본 스탯 5개 공개 완료
부가 옵션 1 공개: 신속한 - 쿨타임 -8%
부가 옵션 2 공개: 명중의 - 발사체 속도 +12%
부가 옵션 3 잠김

획득 결과:
신속한 화염 미사일 지팡이 명중의
적용 affix: 2개
```

### 6.3 장착 슬롯

플레이어는 최대 3개의 무기 슬롯을 가진다.

```text
Weapon Slot 1
Weapon Slot 2
Weapon Slot 3
Inventory
```

공격 순서:

```text
Slot 1 공격
  -> Slot 1 cooldown 시작
  -> Slot 2 사용 가능하면 공격
  -> Slot 3 사용 가능하면 공격
  -> 다시 Slot 1 검사
```

각 무기의 공격 빈도는 자기 `cooldownMs`에 따른다. 슬롯 순서는 플레이어가 wave 사이에 자유롭게 바꿀 수 있다.

- 부가 옵션 미공개: 비교하지 않음
- 획득 전에는 `예상 성능`이 아니라 `공개된 정보`만 보여줌

## 10. 밸런싱 기준

### 10.1 코인 경제

목표:

- 기본 스탯 전체 공개는 한 wave 보상으로 최소 1개 후보에서 가능해야 한다.
- 4개 후보를 전부 깊게 열 수는 없어야 한다.
- 부가 옵션 3개 이상 공개는 의도적인 투자여야 한다.

초기 권장값:

| 구간 | wave 보상 coin | 기본 스탯 전체 비용 | 부가 옵션 1-2 비용 |
| --- | ---: | ---: | ---: |
| Wave 3-5 | 25-35 | 25-30 | 26 |
| Wave 6-10 | 45-65 | 32-42 | 32-44 |
| Wave 11+ | 70-110 | 45-60 | 44-70 |

### 10.2 후보 보정

완전 랜덤만 쓰면 4개 후보가 모두 무의미해질 수 있다.

보정 규칙:

- 빈 무기 슬롯이 있으면 최소 1개는 저렴한 기본 스탯 비용으로 생성
- 같은 공격 방식만 반복 등장하지 않게 패널 안에서 다양성 보장
- 현재 장착 원소가 한쪽으로 치우치면 다른 원소 후보 1개 이상 가중
- 보스 wave 후에는 Rare 이상 후보 확률 증가
- 이전 감정 phase에서 획득하지 못했다면 다음 phase의 기본 비용 일부 완화

### 10.3 위험과 대응

| 위험 | 대응 |
| --- | --- |
| 4패널이 너무 복잡함 | 기본 스탯 row 순서 고정, 잠김 row는 짧게 표시 |
| 코인이 너무 적어 감정이 무의미함 | 최소 1개 후보의 기본 스탯 공개 비용 보장 |
| 코인이 너무 많아 선택이 사라짐 | 부가 옵션 비용을 후반으로 갈수록 크게 증가 |
| 이동 중 coin이 빠져 억울함 | 정지 판정 이후에만 coin 소모 |
| 패널 이탈 초기화가 가혹함 | 이미 공개된 row는 유지하고 진행 중 row만 초기화 |
| 미공개 옵션이 손해처럼 느껴짐 | 공개한 부가 옵션만 붙는다는 규칙을 UI에 명확히 표시 |

## 11. 구현 단계

### Phase 1: 4패널 감정 MVP

- wave clear 후 `shopIntermission` 상태 추가
- 전투 정지, 적 소환 정지, 위험 판정 정리
- 무기 후보 `WeaponInstance` 4개 생성
- `RevealPanel` 4개를 2x2로 표시
- 기본 스탯 row를 고정 순서로 생성
- 캐릭터 위치와 패널 영역 매칭 구현
- 정지 상태에서만 coin 1개씩 소모하며 게이지 충전
- 기본 스탯 전체 공개 시 아이템 획득 가능 처리

완료 기준:

- wave 3 이후 4개의 미감정 아이템 패널이 등장한다.
- shopIntermission 동안 적이 새로 소환되지 않고 자동 공격이 발동하지 않는다.
- 플레이어가 패널 안에서 멈추면 `???` 게이지가 coin을 소모하며 찬다.
- 기본 스탯을 모두 공개하면 해당 아이템을 획득할 수 있다.

### Phase 2: 부가 옵션 감정

- 부가 옵션 row 4-6개 생성
- 부가 옵션 row 비용 증가 곡선 적용
- 언락한 부가 옵션만 최종 아이템에 적용
- coin 부족 시 진행 정지 처리
- 패널 이탈 시 진행 중 row 초기화 처리

완료 기준:

- 기본 스탯 공개 후에도 부가 옵션을 추가로 감정할 수 있다.
- 부가 옵션은 공개한 개수만 실제 아이템에 붙는다.
- coin이 부족하면 게이지가 일부 차다가 멈춘다.
- 패널을 벗어나면 진행 중 row만 초기화된다.

### Phase 3: 장착과 비교

- 공개된 스탯만 현재 장착 무기와 비교
- 획득 후 인벤토리 추가
- 빈 무기 슬롯 즉시 장착
- 기존 무기와 교체
- 슬롯 순서 정비

완료 기준:

- 공개되지 않은 스탯은 비교 UI에서 `???`로 남는다.
- 획득한 무기를 인벤토리에 보관할 수 있다.
- 1-3번 무기 슬롯에 장착하고 순서를 바꿀 수 있다.

### Phase 4: 후보 생성 보정

- wave, 보유 무기, 빈 슬롯 상태 기반 후보 보정
- 너무 낮은 등급 후보 방지
- 같은 공격 방식만 반복되는 상황 방지
- 4개 후보 안에서 원소/공격 방식 다양성 보장
- 보스 wave 후 희귀도와 부가 옵션 수 보정

완료 기준:

- 4개 패널이 무작위지만 불공정하게 느껴지지 않는다.
- 플레이어 상태에 맞는 후보가 적절히 보정된다.
- 최소 1개 후보는 기본 스탯을 모두 열어볼 만한 비용으로 나온다.

### Phase 5: 장기 밸런싱

- wave별 가격과 coin 획득량 조정
- 희귀도 테이블 조정
- 패널별 평균 감정 row 수 분석
- 특정 속성/공격 방식 편중 검토

완료 기준:

- 한 판에서 최소 2개 이상의 빌드 방향이 자연스럽게 형성된다.
- 4개 후보 중 최소 1개는 선택 가치가 있다.
- 감정 운이 나빠도 다음 wave에서 회복 가능한 장치가 작동한다.

## 12. 테스트 계획

단위 테스트:

- 같은 seed에서 같은 4개 패널과 같은 hidden `WeaponInstance`가 생성되는지 확인
- 모든 row가 처음에는 `???`와 게이지로 표시되는지 확인
- 기본 스탯 row 순서가 항상 고정인지 확인
- 부가 옵션 row 수가 4-6개 범위인지 확인
- 정지 상태에서만 coin이 1개씩 소모되는지 확인
- 이동 중에는 coin과 `revealProgress`가 변하지 않는지 확인
- 패널 이탈 시 진행 중 row만 0으로 초기화되는지 확인
- 이미 `revealed`인 row가 유지되는지 확인
- coin 부족 시 coin 0에서 게이지 진행이 멈추는지 확인
- 기본 스탯 전체 공개 전에는 claim이 불가능한지 확인
- claim 시 공개한 부가 옵션만 `WeaponInstance`에 적용되는지 확인

플레이 테스트:

- wave 1-5에서 4패널 감정이 너무 복잡하지 않은지 확인
- 기본 스탯 공개 비용이 너무 부담스럽지 않은지 확인
- 부가 옵션 공개 비용이 충분한 coin sink인지 확인
- 4개 후보를 조금씩 열어보는 선택이 재미있는지 확인
- 한 후보에 몰아 투자하는 선택도 유효한지 확인
- coin이 항상 부족하거나 항상 남아돌지 않는지 확인
