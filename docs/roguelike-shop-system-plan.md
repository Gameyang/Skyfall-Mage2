# Roguelike Shop System Plan

작성일: 2026-06-12  
대상 프로젝트: `F:\Workspace\Skyfall-Mage2`

## 1. 목표

전투 wave를 클리어할 때마다 일정 확률로 여러 종류의 상점이 등장하고, 플레이어가 그중 하나를 선택해 입장하는 로그라이크식 성장 시스템을 만든다.

핵심 목표:

- wave 사이의 짧은 선택 구간을 만들어 전투 리듬을 끊지 않고 성장을 제공한다.
- 매 판 다른 상점과 상품 조합이 나오게 해서 반복 플레이의 변수를 만든다.
- 물약, 무기, 장비 상점이 서로 다른 성장 축을 담당하게 한다.
- 특정 빌드만 강해지는 문제가 생기지 않도록 속성, 장비 슬롯, 회복, 임시 강화가 고르게 출현한다.
- 상점 입장은 선택 사항으로 두되, 입장하지 않는 선택에도 작은 보상을 줘서 강제감을 줄인다.

## 2. 참고 게임에서 가져올 원칙

| 참고 게임 | 가져올 원칙 | Skyfall Mage 적용 방향 |
| --- | --- | --- |
| `Slay the Spire` | 경로 선택, 상점/이벤트/전투 중 하나를 고르는 구조 | wave 클리어 후 2-3개의 상점 후보를 제시하고 한 곳만 입장 |
| `Hades` | 전투 방 사이의 짧은 보상 선택, 임시 강화와 누적 성장의 조합 | 다음 wave에 바로 체감되는 물약/버프와 장기 장비 성장을 함께 제공 |
| `Dead Cells` | 무기/장비 상점 분리, 같은 카테고리 안에서 교체와 강화 선택 | 원소 무기 상점과 장비 슬롯 상점을 명확히 분리 |
| `The Binding of Isaac` | 코인 기반 상점, 회복/소모품/특수 아이템의 리스크 있는 구매 | 부족한 자원을 상점에서 보완하되 가격과 등장 확률로 제어 |
| `Risk of Rain 2` | 멀티샵, 아이템 선택 후 나머지 선택지가 사라지는 방식 | 한 상점 안에서도 1개 무료 선택 또는 고가 상품 선택 후 품절 처리 가능 |
| `Vampire Survivors`류 | 짧은 전투 반복, 빌드 시너지 중심 성장 | 상점 상품을 속성/쿨다운/생존/이동 시너지로 묶어 빠르게 읽히게 함 |

이 시스템은 완전한 마을 상점이 아니라 `wave 사이의 선택형 성장 이벤트`로 설계한다. 플레이어가 오래 머무르며 쇼핑하는 것보다, 현재 빌드에 필요한 축을 빠르게 고르는 경험이 우선이다.

## 3. 전체 흐름

```text
Wave Clear
  -> 기본 보상 지급
  -> 상점 후보 0-3개 생성
  -> 전투 정지, 적 소환 정지
  -> 밤하늘 전장 위에 비행선 상점 후보 또는 보급 매대 등장
  -> 플레이어가 캐릭터를 원하는 상점/상품 위치로 이동
  -> 상품 구매, 강화, 교체, 재추첨
  -> 비행선 출발 지점에서 정비 완료
  -> 다음 wave 시작
```

### 3.1 상점 등장 타이밍

초기 권장안:

| 구간 | 등장 규칙 | 의도 |
| --- | --- | --- |
| Wave 1-2 | 상점 미등장 또는 튜토리얼 물약 상점 1회 | 초반 전투 학습 우선 |
| Wave 3-5 | wave 클리어 시 60% 확률로 상점 후보 2개 | 상점 시스템 소개 |
| Wave 6-10 | wave 클리어 시 80% 확률로 상점 후보 2-3개 | 빌드 선택 본격화 |
| Wave 11+ | wave 클리어 시 100% 확률로 상점 후보 3개 | 로그라이크 성장 루프 고정 |
| 보스/엘리트 wave 후 | 항상 상점 후보 3개, 희귀 상품 확률 증가 | 큰 전투 후 확실한 보상감 |

상점은 매 wave마다 무조건 들어가야 하는 메뉴가 아니라, `어떤 성장 방향을 이번 판에 선택할 것인가`를 묻는 이벤트다.

### 3.2 후보 선택 방식

wave 클리어 후 별도 상점 화면으로 전환하지 않는다. 현재 전장과 밤하늘 배경을 유지한 채, 화면 안에 비행선 상점 후보 또는 보급 매대가 실제 오브젝트처럼 등장한다.

- 기본 후보 수: 2개
- 중반 이후 후보 수: 3개
- 플레이어는 캐릭터를 움직여 한 곳만 선택 가능
- 선택하지 않은 비행선/매대는 이륙하거나 닫힘
- 스킵은 `다음 wave 시작 지점` 또는 `보급 거절 지점`으로 이동해 선택
- 스킵 시 `소량 회복`, `소량 코인`, `다음 상점 할인권` 중 하나 지급

후보 예시:

```text
[물약 비행선] [불 무기 비행선] [머리 장비 비행선]
```

후보 오브젝트에는 상세 상품을 모두 보여주지 않고, 상점 타입과 대략적인 성격만 보여준다. 특정 비행선 근처로 이동해 선택하면 그 비행선이 정박하고 실제 상품 3-5개가 매대에 펼쳐진다.

### 3.3 인월드 상점 phase

상점 phase는 `전투 화면 위에 얹힌 조작 가능한 정비 구간`이다.

상태 규칙:

- 플레이어 이동은 유지한다.
- 자동 공격은 멈춘다.
- 무기 쿨타임은 진행하지 않는다.
- 적 소환은 멈춘다.
- 남은 적, 위험 투사체, 피해 장판은 정리한다.
- 카메라는 전투 화면을 유지하되 비행선과 상품 매대가 잘 보이도록 살짝 보정할 수 있다.
- 구매와 장착 정비가 끝나면 플레이어가 다음 wave 시작 위치로 이동하거나 확인 입력을 한다.

비행선 상점 연출:

- 밤하늘 배경 위쪽에서 비행선이 내려온다.
- 비행선 아래에 상품 화물칸, 매대, 로프에 매달린 상자가 열린다.
- 각 상품은 플레이어가 접근할 수 있는 월드 좌표를 가진다.
- 상품 근처에 접근하면 간단한 가격/등급/효과 패널이 뜬다.
- 구매 성공 후 해당 매대는 품절 상태로 남는다.

이 구조는 상점이 별도 UI가 아니라 `wave 사이에 전장으로 찾아오는 보급선`처럼 느껴지게 한다.

## 4. 상점 타입

상점은 크게 3계열, 세부 7종으로 나눈다.

| 계열 | 세부 상점 | 역할 |
| --- | --- | --- |
| 물약 | 물약 상점 | 회복, 자원 보충, 다음 wave 임시 강화 |
| 무기 | 불 무기 상점 | 광역 피해, 지속 피해, 폭발 |
| 무기 | 물 무기 상점 | 둔화, 빙결, 제어, 안정성 |
| 무기 | 전기 무기 상점 | 연쇄 피해, 순간 폭딜, 쿨다운, 기동 |
| 장비 | 머리 장비 상점 | 마나, 쿨다운, 스킬, 시야/감지 |
| 장비 | 상의 장비 상점 | 체력, 방어, 피해 감소, 생존 |
| 장비 | 하의 장비 상점 | 이동, 회피, 대시, 경제/수집 |

하의 장비는 현재 에셋 상황상 바지와 신발 장비를 함께 포함하는 슬롯으로 시작한다. 추후 장비 슬롯을 세분화할 경우 `하의`와 `신발`을 분리할 수 있다.

## 5. 물약 상점

### 5.1 역할

물약 상점은 전투 중 손상된 상태를 회복하고, 다음 wave를 버틸 수 있는 임시 강화를 제공한다. 플레이어가 빌드를 완전히 바꾸는 곳이 아니라, 현재 판의 위험을 완화하는 안전장치다.

### 5.2 상품 카테고리

| 카테고리 | 예시 상품 | 효과 방향 |
| --- | --- | --- |
| HP 회복 | 작은 HP 물약, 큰 HP 물약 | 즉시 체력 회복 |
| MP 회복 | 작은 MP 물약, 큰 MP 물약 | 즉시 마나 회복 |
| 재생 | 생명 재생 물약, 마나 재생 물약 | 다음 wave 동안 초당 회복 |
| 공격 강화 | 공격력 물약, 치명 물약 | 다음 wave 동안 피해량 증가 |
| 속도 강화 | 이동 속도 물약, 시전 속도 물약 | 다음 wave 동안 이동/공격 템포 증가 |
| 쿨다운 강화 | 쿨다운 감소 물약 | 다음 wave 동안 스킬 재사용 시간 감소 |
| 긴급 생존 | 보호막 물약, 사망 방지 물약 | 다음 wave에서 1회 보호 효과 |

현재 가져온 에셋과 연결 가능한 예시:

- `hp_potion_1` - `hp_potion_5`
- `mp_potion_1` - `mp_potion_5`
- `haste_potion_1` - `haste_potion_3`
- `speed_potion_1` - `speed_potion_3`
- `cooldown_potion_1` - `cooldown_potion_3`
- `instant_attack_buff`
- `instant_speed_buff`

### 5.3 상품 구성 규칙

물약 상점 기본 재고:

- 회복 물약 1-2개
- 임시 강화 물약 1-2개
- 희귀 생존 물약 0-1개
- wave 8 이후에는 복합 물약 1개 등장 가능

초기 예시:

| 상품 | 가격 | 효과 |
| --- | ---: | --- |
| 작은 HP 물약 | 12 coin | 최대 체력의 25% 회복 |
| 중간 MP 물약 | 16 coin | 최대 마나의 40% 회복 |
| 신속 물약 | 24 coin | 다음 wave 동안 이동 속도 +18% |
| 집중 물약 | 32 coin | 다음 wave 동안 쿨다운 -12% |
| 보호막 물약 | 45 coin | 다음 wave 시작 시 최대 체력 30% 보호막 |

### 5.4 밸런싱 원칙

- 체력이 낮은 플레이어에게는 HP 회복 상품 확률을 올린다.
- MP 기반 무기를 장착 중이면 MP 회복과 쿨다운 물약 확률을 올린다.
- 물약 효과는 강하게 주되 대부분 `다음 wave 동안`만 유지한다.
- 물약 상점만 반복해서 들어가도 장기 성장에서는 무기/장비 상점보다 뒤처지게 만든다.

## 6. 무기 상점

### 6.1 공통 역할

무기 상점은 플레이어의 공격 방식을 결정하는 핵심 성장처다. 세부 상점은 `불`, `물`, `전기` 3가지 속성으로 나누고, 해당 속성 안에서 무기 획득, 강화, 교체, 변형을 제공한다.

무기 상점 공통 기능:

- 해당 속성 신규 무기 판매
- 보유 중인 같은 속성 무기 강화
- 같은 속성 무기의 공격 방식 변형
- 속성 전용 강화 재료 판매
- 낮은 등급 무기를 상위 등급으로 승급
- 다른 속성 무기와 섞는 융합 옵션은 중후반 이후 해금

### 6.2 불 무기 상점

역할: 높은 피해량, 광역 폭발, 지속 화상.

플레이 감각:

- 적이 몰려오는 wave에서 강하다.
- 단일 대상 안정성은 전기보다 낮고, 제어력은 물보다 낮다.
- 환경 효과가 들어오면 연소, 증발, 폭발과 연결하기 좋다.

상품 예시:

| 상품 | 타입 | 효과 방향 |
| --- | --- | --- |
| 화염 지팡이 | 신규 무기 | 기본 불 projectile 발사 |
| 화염 부채 | 신규 무기 | 부채꼴 범위 공격 |
| 화염 미사일 | 신규 무기 | 느리지만 강한 추적 폭발 |
| 메테오 지팡이 | 희귀 무기 | 지연 후 큰 광역 피해 |
| 화염방사 강화 | 무기 변형 | 짧은 사거리 지속 피해 |
| 화상 증폭 룬 | 강화 | 화상 피해량 증가 |

현재 에셋 연결 후보:

- `fire_staff`
- `fire_fan_staff`
- `fire_missile_staff`
- `fire_meteor_staff`
- `flamethrower`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 불씨 확산 | 화상 적 사망 시 주변에 작은 폭발 |
| 과열 | 공격력 증가, MP 소모 증가 |
| 잔불 | 화상 지속 시간 증가 |
| 폭열 | 폭발 반경 증가, projectile 속도 감소 |

### 6.3 물 무기 상점

역할: 둔화, 빙결, 제어, 생존 안정성.

플레이 감각:

- 빠른 적과 돌진형 적을 상대하기 좋다.
- 피해량은 불보다 낮지만 안정성이 높다.
- 물/빙결 계열로 시작하고, 추후 물질 시뮬레이션과 연결하면 젖음, 동결, 미끄러짐으로 확장할 수 있다.

상품 예시:

| 상품 | 타입 | 효과 방향 |
| --- | --- | --- |
| 눈보라 지팡이 | 신규 무기 | 범위 둔화 장판 |
| 얼음 회오리 | 신규 무기 | 적을 밀어내며 지속 피해 |
| 물방울 탄 | 신규 무기 | 낮은 피해, 높은 둔화 |
| 서리 핵 | 강화 재료 | 빙결 확률 증가 |
| 냉각 개조 | 무기 변형 | 공격 속도 감소, 제어력 증가 |

현재 에셋 연결 후보:

- `blizzard_staff`
- `ice_tornado_staff`
- `snowball`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 냉기 축적 | 같은 적을 여러 번 맞히면 빙결 |
| 서리 파편 | 빙결 적 처치 시 파편 발사 |
| 습기 전도 | 젖은 적에게 전기 피해를 더 잘 전달 |
| 얕은 물길 | projectile이 지나간 자리에 짧은 둔화 영역 생성 |

### 6.4 전기 무기 상점

역할: 연쇄 피해, 순간 폭딜, 쿨다운, 기동.

플레이 감각:

- 적이 적당히 뭉쳐 있을 때 효율이 높다.
- 단일 대상 burst와 다중 대상 chain 사이의 선택이 중요하다.
- 이동/대시와 결합해 고숙련 플레이어에게 보상이 크다.

상품 예시:

| 상품 | 타입 | 효과 방향 |
| --- | --- | --- |
| 연쇄 번개 지팡이 | 신규 무기 | 가까운 적에게 번개 전이 |
| 낙뢰 지팡이 | 신규 무기 | 높은 피해의 단일 낙뢰 |
| 천둥 대시 지팡이 | 신규 무기 | 이동기와 공격 결합 |
| 충전 코일 | 강화 재료 | 일정 시간마다 다음 공격 강화 |
| 과전류 개조 | 무기 변형 | 연쇄 횟수 증가, 피해량 감소 |

현재 에셋 연결 후보:

- `chain_lightning_staff`
- `lightning_strike_staff`
- `thunder_dash_staff`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 전도 | 젖은 적에게 chain 우선 적용 |
| 축전 | 공격하지 않는 동안 다음 공격 피해 증가 |
| 정전기 | 근접한 적에게 주기적 소형 피해 |
| 과충전 | 쿨다운 감소, 일정 확률로 self slow |

### 6.5 무기 상점 재고 규칙

각 원소 무기 상점 기본 재고:

- 신규 무기 1-2개
- 보유 무기 강화 1개
- 속성 재료 1개
- 희귀 변형 또는 승급 0-1개

보유 무기와 상점 속성이 맞을 때:

- 강화 상품 확률 증가
- 같은 무기 중복 구매 시 `강화 재료` 또는 `승급`으로 변환
- 이미 장착한 무기보다 낮은 등급의 신규 무기는 가격 할인 또는 제외

보유 무기와 상점 속성이 다를 때:

- 신규 무기 확률 증가
- 속성 전환을 쉽게 하기 위해 첫 구매 가격 할인
- 단, 매번 속성을 갈아타도 손해가 너무 작지 않도록 강화 누적치는 속성별로 분리

## 7. 장비 상점

### 7.1 공통 역할

장비 상점은 플레이어의 기본 능력치와 패시브를 안정적으로 올리는 장기 성장처다. 무기 상점이 공격 방식의 변화라면, 장비 상점은 생존성, 자원 운용, 이동, 경제를 담당한다.

장비 공통 기능:

- 해당 슬롯 신규 장비 판매
- 보유 장비 강화
- 장비 옵션 재련
- 장비 등급 상승
- 같은 슬롯 내 교체
- 세트 효과 또는 태그 시너지 부여

장비 기본 등급:

```text
Common -> Uncommon -> Rare -> Epic -> Legendary
```

강화 단계:

```text
+0 -> +1 -> +2 -> +3 -> +4 -> +5
```

초기 구현에서는 등급과 강화 단계를 동시에 복잡하게 쓰지 말고, 먼저 `등급` 중심으로 만들고 이후 강화 단계를 붙이는 편이 안전하다.

### 7.2 머리 장비 상점

역할: 마나, 쿨다운, 스킬, 감지.

상품 예시:

| 상품 | 효과 방향 |
| --- | --- |
| 마나 재생 모자 | MP 재생 증가 |
| 거대 마나 왕관 | 최대 MP 증가 |
| 정적 두건 | 가만히 있을 때 쿨다운 감소 |
| 생명 교환 투구 | HP를 소모해 MP 회복 |
| 집중 렌즈 | 스킬 피해 또는 치명 확률 증가 |

현재 에셋 연결 후보:

- `mana_regen_cap`
- `vast_mana_crown`
- `stillness_hood`
- `vital_exchange_helm`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 지혜 | 최대 MP 증가 |
| 명상 | MP 재생 증가 |
| 주문 가속 | 쿨다운 감소 |
| 예지 | 다음 상점 상품 1개 미리보기 |

### 7.3 상의 장비 상점

역할: 최대 체력, 방어, 피해 감소, 위기 생존.

상품 예시:

| 상품 | 효과 방향 |
| --- | --- |
| 절박한 갑옷 | 체력이 낮을수록 방어 증가 |
| 거대 생명 갑옷 | 최대 HP 증가 |
| 마나 맥동 갑옷 | MP 사용 시 보호막 획득 |
| 보폭 재생 갑옷 | 이동 거리 기반 체력 회복 |
| 원소 저항 로브 | 특정 속성 피해 감소 |

현재 에셋 연결 후보:

- `desperation_mail`
- `vast_vital_armor`
- `mana_pulse_armor`
- `stride_regen_armor`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 단단함 | 받는 피해 감소 |
| 활력 | 최대 HP 증가 |
| 응급 보호 | 낮은 HP에서 보호막 발동 |
| 흡수 | 일정 피해를 MP 회복으로 전환 |

### 7.4 하의 장비 상점

역할: 이동 속도, 대시, 회피, 수집, 경제.

현재 하의 슬롯은 신발/장화류까지 포함하는 이동 장비 슬롯으로 운영한다.

상품 예시:

| 상품 | 효과 방향 |
| --- | --- |
| 점멸 장화 | 대시 또는 순간 이동 강화 |
| 탐욕 장화 | 코인 획득량 증가 |
| 물결 장화 | 물/빙결 지형에서 이동 보너스 |
| 순풍 장화 | 이동 속도와 회피 증가 |
| 수집 바지 | 아이템 pickup radius 증가 |

현재 에셋 연결 후보:

- `blinkstep_boots`
- `greedstep_boots`
- `tidewalker_boots`
- `slipstream_boots`

강화 예시:

| 강화 | 효과 |
| --- | --- |
| 민첩 | 이동 속도 증가 |
| 회피 | 피격 회피 확률 또는 dash 무적 시간 증가 |
| 자석 | 아이템 획득 반경 증가 |
| 행운 | 상점 할인 또는 희귀 상품 확률 소폭 증가 |

### 7.5 장비 상점 재고 규칙

각 슬롯 상점 기본 재고:

- 신규 장비 1-2개
- 보유 장비 강화 1개
- 옵션 재련 1개
- 희귀 장비 0-1개

보유 슬롯이 비어 있을 때:

- 신규 장비 2개 이상 보장
- 가장 낮은 등급 상품 1개는 저가로 제공

보유 장비가 있을 때:

- 강화 또는 재련 상품 1개 보장
- 기존 장비보다 명확히 낮은 상품은 제외
- 교체 상품은 `공격형`, `방어형`, `경제형`처럼 방향이 다르게 나오게 한다.

## 8. 랜덤 생성 규칙

### 8.1 상점 후보 가중치

초기 기본 가중치:

| 상점 계열 | 가중치 |
| --- | ---: |
| 물약 상점 | 25 |
| 무기 상점 전체 | 45 |
| 장비 상점 전체 | 30 |

세부 상점 가중치:

| 세부 상점 | 기본 가중치 |
| --- | ---: |
| 물약 | 25 |
| 불 무기 | 15 |
| 물 무기 | 15 |
| 전기 무기 | 15 |
| 머리 장비 | 10 |
| 상의 장비 | 10 |
| 하의 장비 | 10 |

### 8.2 동적 보정

상점 후보는 완전 랜덤이 아니라 플레이어 상태를 반영한다.

| 조건 | 보정 |
| --- | --- |
| 현재 HP 35% 이하 | 물약 상점 가중치 +20 |
| 무기 슬롯이 비어 있음 | 무기 상점 가중치 +20 |
| 특정 원소 무기를 2개 이상 보유 | 해당 원소 상점 가중치 +10 |
| 장비 슬롯이 비어 있음 | 해당 슬롯 장비 상점 가중치 +15 |
| 같은 상점이 직전 2회 등장 | 해당 상점 가중치 -50 |
| 보스 wave 후 | 무기/장비 상점 희귀 상품 확률 증가 |

### 8.3 중복 방지

- 같은 후보 화면 안에 동일 세부 상점은 중복 등장하지 않는다.
- 직전 입장한 상점은 다음 후보에서 등장 확률을 낮춘다.
- 3회 연속 물약 상점만 고르는 상황을 막기 위해 장기 성장 상점 보정치를 올린다.
- 3회 이상 무기 상점이 나오지 않았다면 다음 후보에 무기 상점 1개를 보장한다.
- 특정 슬롯 장비가 비어 있으면 해당 슬롯 상점이 너무 늦게 나오지 않게 pity counter를 둔다.

## 9. 가격과 경제

### 9.1 화폐

초기 화폐는 `coin` 하나로 시작한다. 추후 확장 시 `gem`, `coupon`, `membership` 같은 자산을 특수 경제로 연결할 수 있다.

현재 에셋 연결 후보:

- `coin`
- `coin_pack_small`
- `coin_pack_large`
- `gem`
- `coupon_pack`
- `vip_membership`
- `vvip_membership`

### 9.2 가격 공식

권장 기본 공식:

```text
price = basePrice * rarityMultiplier * waveMultiplier * shopModifier * discountModifier
```

초기 값:

| 요소 | 값 |
| --- | ---: |
| Common multiplier | 1.0 |
| Uncommon multiplier | 1.35 |
| Rare multiplier | 1.9 |
| Epic multiplier | 2.8 |
| Legendary multiplier | 4.2 |
| Wave multiplier | `1 + waveIndex * 0.06` |
| 할인 상점 modifier | 0.85 |
| 고급 상점 modifier | 1.15 |

### 9.3 구매 제한

- 한 상점에서 모든 상품을 다 살 수 있게 만들면 선택이 약해진다.
- 기본은 3-5개 재고 중 1-3개만 살 수 있는 경제량을 목표로 한다.
- 희귀 상품은 비싸게 두되, 플레이어가 돈을 모았다면 명확히 살 수 있어야 한다.
- 초반 첫 상점에는 반드시 구매 가능한 저가 상품 1개를 보장한다.

## 10. 희귀도와 상품 풀

### 10.1 wave 구간별 희귀도

| Wave 구간 | Common | Uncommon | Rare | Epic | Legendary |
| --- | ---: | ---: | ---: | ---: | ---: |
| 1-3 | 75% | 25% | 0% | 0% | 0% |
| 4-7 | 55% | 35% | 10% | 0% | 0% |
| 8-12 | 40% | 38% | 18% | 4% | 0% |
| 13-18 | 28% | 36% | 25% | 10% | 1% |
| 19+ | 20% | 32% | 30% | 15% | 3% |

보스 wave 후에는 Rare 이상 확률을 1단계 올린다.

### 10.2 상품 풀 구조

상품은 `shopType`, `tags`, `rarity`, `minWave`, `requires`, `excludes`를 가진다.

예시:

```js
{
  id: 'fire_meteor_staff',
  shopType: 'weapon:fire',
  rarity: 'rare',
  minWave: 7,
  tags: ['weapon', 'fire', 'aoe', 'burst'],
  price: 90,
  effect: {
    type: 'grantWeapon',
    weaponId: 'fire_meteor_staff',
  },
}
```

장비 강화 예시:

```js
{
  id: 'upgrade_headgear_cooldown',
  shopType: 'equipment:head',
  rarity: 'uncommon',
  minWave: 4,
  tags: ['equipmentUpgrade', 'head', 'cooldown'],
  price: 54,
  effect: {
    type: 'upgradeEquippedSlot',
    slot: 'head',
    stat: 'cooldownReduction',
    value: 0.06,
  },
}
```

## 11. 상점 내부 UX

### 11.1 화면 구성

상점은 전체 화면 메뉴가 아니라 전장 안에 배치되는 인월드 UI다. 화면 전체를 덮지 않고, 플레이어가 접근한 상품만 자세히 보여준다.

필수 정보:

- 상점 이름
- 현재 코인
- 상품 3-5개를 나타내는 월드 오브젝트
- 상품 등급
- 가격
- 현재 장착 아이템과 비교되는 핵심 수치
- 접근 시 구매 버튼
- 비행선의 재추첨 장치
- 다음 wave 시작 지점

상품 카드 표시 정보:

```text
[아이콘] 화염 미사일 지팡이
Rare / Fire Weapon
느린 추적탄을 발사하고 폭발 피해를 줌
90 coin
[구매]
```

상호작용 흐름:

```text
플레이어가 상품 매대 근처로 이동
  -> 상품 요약 패널 표시
  -> coin 부족/구매 가능 상태 표시
  -> 구매 확인 입력
  -> 구매 성공 시 품절 처리
```

상품 구매는 단순 충돌로 즉시 처리하지 않는다. 플레이어가 상품 옆을 지나가다 실수로 coin을 쓰지 않도록, 접근 후 확인 입력을 요구한다.

### 11.2 재추첨

재추첨은 로그라이크 상점의 변수를 조절하는 장치다.

- 상점당 기본 1회 가능
- 첫 재추첨 가격: 10 coin
- 두 번째 이후 가격: 25, 45, 70처럼 증가
- 재추첨 후 같은 상품이 다시 나오지 않게 한다.
- `행운`, `멤버십`, `쿠폰` 효과로 재추첨 비용을 낮출 수 있다.

### 11.3 무료 선택 이벤트

일부 상점은 구매가 아니라 `하나만 선택`하는 구조로 등장할 수 있다.

예시:

```text
전기 무기 상점 - 과충전 이벤트
세 개의 전기 강화 중 하나를 무료로 선택한다.
선택하지 않은 강화는 사라진다.
```

이 방식은 `Risk of Rain 2`의 멀티샵처럼 선택의 밀도를 높이는 데 좋다.

## 12. 성장 시너지

### 12.1 속성 시너지

| 조합 | 효과 방향 |
| --- | --- |
| 불 + 불 | 화상, 폭발, 광역 피해 강화 |
| 물 + 물 | 둔화, 빙결, 생존 안정성 강화 |
| 전기 + 전기 | chain, burst, 쿨다운 강화 |
| 물 + 전기 | 젖은 적에게 전도 피해 |
| 불 + 물 | 증기 폭발, 시야 방해, 냉각/과열 변형 |
| 불 + 전기 | 과부하 폭발, 짧은 burst 강화 |

초기 구현에서는 단일 속성 시너지만 먼저 적용하고, 혼합 속성은 콘텐츠가 충분해진 뒤 추가한다.

### 12.2 장비-무기 시너지

| 장비 슬롯 | 무기와의 연결 |
| --- | --- |
| 머리 | MP, 쿨다운, 스킬 피해로 모든 무기 운용 강화 |
| 상의 | 생존성을 올려 짧은 사거리 무기 운용 보조 |
| 하의 | 이동/대시를 올려 위치 선정과 회피 강화 |

예시:

- 불 무기 + 상의 `절박한 갑옷`: 근접 화염방사 운용 가능
- 물 무기 + 하의 `물결 장화`: 둔화 장판 안에서 안정적으로 이동
- 전기 무기 + 머리 `마나 재생 모자`: chain lightning 유지력 증가

## 13. 밸런싱 위험과 대응

| 위험 | 대응 |
| --- | --- |
| 물약 상점이 너무 약해서 무시됨 | 임시 강화 수치를 강하게 주고 가격을 낮춤 |
| 물약 상점만 반복하면 생존이 과도해짐 | 장기 성장 능력은 무기/장비보다 낮게 제한 |
| 특정 원소만 압도적으로 강함 | 원소별 역할을 분리하고 enemy/wave에서 상성 상황을 순환 |
| 상점 운이 나쁘면 빌드가 망함 | pity counter와 상태 기반 가중치 적용 |
| 코인이 너무 많아 선택이 사라짐 | 재추첨, 강화, 희귀 상품으로 고가 sink 제공 |
| 코인이 너무 적어 상점이 무의미함 | 첫 상품 저가 보장, 스킵 보상, 할인 이벤트 제공 |
| 장비 슬롯이 많아 이해가 어려움 | 초기에는 머리/상의/하의 3슬롯만 운영 |
| 상품 설명이 길어짐 | 효과는 한 줄 요약, 상세 수치는 hover 또는 비교 패널에 표시 |

## 16. 구현 단계

### Phase 1: 상점 후보와 물약 상점

- wave clear 후 `shopIntermission` 상태 추가
- shopIntermission 동안 자동 공격, 적 소환, 위험 판정 정지
- 밤하늘 전장 위에 비행선 상점 후보 오브젝트 추가
- 상품 매대와 접근 반경 기반 구매 UI 추가
- 물약 상점 상품 풀 구현
- coin 가격과 구매 처리 구현
- 다음 wave 임시 buff 처리 구현

완료 기준:

- wave 3 이후 전장 위에 비행선 상점 후보가 등장한다.
- 플레이어가 캐릭터를 움직여 물약 상점 매대에 접근하고 상품을 구매할 수 있다.
- shopIntermission 동안 적이 새로 소환되지 않고 자동 공격이 발동하지 않는다.
- 구매한 물약 효과가 즉시 또는 다음 wave에 적용된다.

### Phase 2: 무기 상점

- 불/물/전기 상점 타입 추가
- 무기 획득 효과 추가
- 보유 무기 강화 효과 추가
- 속성별 상품 풀과 희귀도 적용

완료 기준:

- 상점에서 원소 무기를 획득할 수 있다.
- 같은 속성 무기를 강화할 수 있다.
- 각 원소 상점의 체감 역할이 다르다.

### Phase 3: 장비 상점

- 머리/상의/하의 슬롯 모델 추가
- 장비 획득, 교체, 강화 처리 추가
- 장비별 패시브 효과 적용
- 현재 장착 장비와 상품 비교 UI 추가

완료 기준:

- 각 장비 슬롯에 아이템을 장착할 수 있다.
- 슬롯별 상점에서 해당 장비만 등장한다.
- 장비 강화가 전투 수치에 반영된다.

### Phase 4: 재추첨, 할인, 동적 가중치

- 상점 재추첨 구현
- 비행선의 재추첨 장치 월드 오브젝트 구현
- HP/무기/장비 상태 기반 후보 보정
- 중복 방지와 pity counter 구현
- 스킵 보상 구현

완료 기준:

- 상점 후보가 무작위지만 불공정하게 느껴지지 않는다.
- 플레이어 상태에 맞는 상점이 적절히 보정된다.
- 스킵 위치 또는 다음 wave 시작 지점이 명확한 대안으로 작동한다.

### Phase 5: 장기 밸런싱

- wave별 가격과 코인 획득량 조정
- 희귀도 테이블 조정
- 상점별 평균 구매 수 분석
- 특정 속성/장비 슬롯 편중 검토

완료 기준:

- 한 판에서 최소 2개 이상의 빌드 방향이 자연스럽게 형성된다.
- 물약, 무기, 장비 상점 모두 선택 가치가 있다.
- 상점 운이 나빠도 회복 가능한 장치가 작동한다.

## 17. 테스트 계획

단위 테스트:

- 같은 seed에서 같은 상점 후보가 생성되는지 확인
- 후보 안에 같은 세부 상점이 중복되지 않는지 확인
- HP가 낮을 때 물약 상점 가중치가 올라가는지 확인
- 장비 슬롯이 비어 있을 때 해당 슬롯 상점이 보정되는지 확인
- 가격 공식이 wave와 희귀도에 따라 증가하는지 확인
- 구매 후 coin 차감과 상품 품절이 적용되는지 확인

플레이 테스트:

- wave 1-5에서 상점이 너무 복잡하지 않은지 확인
- wave 6-10에서 무기 속성 선택이 체감되는지 확인
- wave 10+에서 장비 강화가 생존에 충분히 기여하는지 확인
- 물약만 사는 플레이가 장기 성장에서 손해인지 확인
- 무기만 사는 플레이가 생존 위험을 감수하는 선택인지 확인
- 코인이 항상 부족하거나 항상 남아돌지 않는지 확인

## 18. 우선순위 결론

가장 먼저 구현할 조합:

1. wave clear 후 상점 후보 2개 생성
2. 물약 상점
3. 불/물/전기 무기 상점
4. 머리/상의/하의 장비 상점
5. 재추첨과 동적 가중치

초기 버전에서는 상점 종류를 모두 만들되, 각 상점의 상품 수는 작게 시작하는 편이 좋다. 상점 시스템의 재미는 상품 개수보다 `이번 wave 이후 어떤 성장 방향을 고를지`가 명확히 드러날 때 먼저 생긴다.
